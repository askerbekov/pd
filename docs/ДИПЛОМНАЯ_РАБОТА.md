## 1. Введение

Современный сайт строительной компании премиум-класса — это не просто витрина с информацией о проектах. Это полноценный маркетинговый инструмент, который должен:

- мгновенно вызывать доверие за счёт визуального уровня дизайна;
- быстро загружаться и плавно работать на любых устройствах;
- собирать лиды (заявки) через удобные формы;
- передавать ценности бренда и эмоцию роскоши.

В качестве прототипа взят реально работающий сайт компании **BSK Capital Group** ([capital-group.kg](https://capital-group.kg)) — лидера в строительстве клубных домов premium-класса в Кыргызстане. Цель работы — спроектировать и реализовать улучшенный аналог сайта на чистых веб-технологиях, без зависимости от тяжёлых фреймворков.

## 4. Выбор технологий и обоснование

| Технология | Версия | Зачем |
|-----------|--------|-------|
| **HTML5** | Living Standard | Семантическая разметка (`<header>`, `<section>`, `<article>`, `<details>`), валидируемая W3C |
| **CSS3** | Modules | Сетка Grid, Flexbox, переменные `--var`, `clamp()`, `backdrop-filter`, `aspect-ratio` |
| **JavaScript** | ES2020+ | Модули `import/export`, `async/await`, деструктуризация, `Intersection Observer API` |
| **Google Fonts** | — | Cormorant Garamond (заголовки) + Manrope (текст) |
| **OpenStreetMap** | embed | Карта офиса без API-ключа Google Maps |
| **Unsplash** | CDN | Бесплатные стоковые фото архитектуры под CC0 |

### 4.1 Почему Vanilla JS, а не React?

1. **Размер бандла.** Vanilla-вариант весит ~15 КБ JS вместо ~150 КБ React + ReactDOM.
2. **Скорость загрузки.** Нет шага hydration — браузер рендерит HTML сразу.
3. **Простота поддержки.** Любой junior может прочитать код без знаний фреймворка.
4. **Цель проекта.** Сайт-визитка не требует управления сложным состоянием.
5. **Демонстрация фундамента.** Дипломная работа должна показать владение **базовыми технологиями**, а не модными библиотеками.

### 4.2 Почему модули ES6?

ES-модули (`<script type="module">`) — нативная возможность браузеров с 2017 года. Они дают:

- **изоляцию областей видимости** (нет глобальных утечек);
- **отложенное выполнение** (`defer` по умолчанию);
- **код-сплиттинг** — каждый модуль грузится отдельно;
- **простую навигацию** — каждая фича в своём файле.

```js
import { initModal } from './modules/modal.js';
```

### 4.3 Почему CSS-переменные?

Переменные CSS (`--accent: #c0a060`) — это **дизайн-токены**, позволяющие:

- менять всю палитру в одном месте;
- легко поддерживать тёмную тему в будущем;
- передавать значения из JavaScript при необходимости.

## 5. Архитектура проекта

### 5.1 Архитектурный шаблон

Применён шаблон **Module Pattern** в сочетании с **функциональным подходом**:

```
main.js (точка входа)
   │
   ├── initPreloader()         ← preloader.js
   ├── initHeader()            ← header.js
   ├── initNavigation()        ← navigation.js
   ├── initHeroSlider()        ← heroSlider.js
   ├── initProjectsFilter()    ← projectsFilter.js
   ├── initModal()             ← modal.js
   ├── initForms()             ← form.js
   ├── initScrollAnimations()  ← scrollAnimations.js
   └── initCounters()          ← counter.js
```

Каждый модуль:
- экспортирует одну init-функцию;
- инкапсулирует своё состояние через замыкания;
- не имеет побочных эффектов вне своего DOM-узла.

### 5.2 Принципы

| Принцип | Реализация |
|---------|-----------|
| **Single Responsibility** | Один модуль = одна фича |
| **DRY** | Утилиты (showToast) выносятся в общие модули |
| **Прогрессивное улучшение** | Сайт работает даже с отключённым JS (форма, навигация) |
| **Graceful degradation** | Fallback'и для отсутствия IntersectionObserver |
| **Performance-first** | throttle через requestAnimationFrame, passive listeners |

## 7. Описание HTML-разметки

### 7.1 Семантическая структура

Использованы все ключевые HTML5-теги:

```html
<header>     — шапка с навигацией
<nav>        — навигационное меню
<section>    — каждая секция страницы
<article>    — карточка проекта (как самостоятельная единица контента)
<details>    — нативный аккордеон без JS
<form>       — формы заявок
<footer>     — подвал
```

### 7.2 BEM-методология

Для именования классов использован **BEM** (Block — Element — Modifier):

```html
<button class="btn btn--primary btn--lg">
       блок ↑   модификатор ↑    модификатор ↑

<div class="hero__content">
        блок ↑   элемент ↑
```

**Преимущества:**
- классы не пересекаются между блоками;
- легко искать стили по имени класса;
- предсказуемая каскадность.

### 7.3 Data-атрибуты как точки расширения

Связь HTML ↔ JS через data-атрибуты:

```html
<button data-modal-open="callback">    → открывает modal-callback
<div data-animate="fade-up">           → анимируется при появлении
<span data-count="800">                → анимированный счётчик до 800
<button data-filter="building">        → фильтр по статусу
```

Это позволяет менять разметку без правок JS.

### 7.4 SEO-теги

```html
<meta name="description" content="...">
<meta name="keywords" content="...">
<meta property="og:title" content="...">
<meta property="og:description" content="...">
<link rel="canonical">
```

## 8. Стилизация (CSS)

### 8.1 Дизайн-токены

```css
:root {
    --bg:           #ffffff;
    --bg-dark:      #0e1115;
    --accent:       #c0a060;    /* шампань-золото */

    --font-display: 'Cormorant Garamond', serif;
    --font-body:    'Manrope', sans-serif;

    --container:    1280px;
    --gutter:       24px;
    --radius:       4px;

    --ease:         cubic-bezier(.22, .61, .36, 1);
}
```

**Palette logic:** премиум-сегмент = тёмный фон + золотой акцент + сдержанная типографика. Никаких ярких цветов и градиентов «вырви глаз».

### 8.2 Шрифтовая пара

| Шрифт | Применение | Почему |
|-------|-----------|--------|
| **Cormorant Garamond** (Serif) | Заголовки, числа | Антиква с засечками → ассоциация с роскошью, классикой |
| **Manrope** (Sans-serif) | Основной текст, UI | Современный гротеск, отличная читаемость |

Сочетание serif + sans-serif — классический приём типографики для премиум-брендов (используют Hermès, Louis Vuitton, Christie's).

### 8.3 Сетка

**CSS Grid** для всех многоколоночных секций:

```css
.projects__grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 32px;
}
```

**Flexbox** для одномерных компонентов (header, кнопки, контролы).

### 8.4 Анимации

| Тип | Реализация | Применение |
|-----|-----------|-----------|
| Переходы | `transition` | hover-эффекты |
| Появление | `transform + opacity` через JS-класс | scroll-анимации |
| Keyframes | `@keyframes` | preloader, стрелка вниз, тосты |

Все анимации используют **cubic-bezier(.22, .61, .36, 1)** — кривую с быстрым стартом и плавным финишем, характерную для премиум-интерфейсов.

### 8.5 Тёмные секции

Секции `.advantages` и `.cta` инвертированы (белый текст на тёмном) — это создаёт **ритм восприятия** при скролле: светло → тёмно → светло.

## 9. JavaScript-модули

### 9.1 main.js — точка входа

```js
import { initModal } from './modules/modal.js';
import { initForms } from './modules/form.js';
// ...

function bootstrap() {
    initPreloader();
    initHeader();
    initNavigation();
    initHeroSlider({ interval: 6500 });
    initProjectsFilter();
    initModal();
    initForms();
    initScrollAnimations();
    initCounters();
}

if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', bootstrap);
} else {
    bootstrap();
}
```

### 9.2 header.js — поведение шапки

Шапка прозрачная над hero и становится белой при скролле > 60px. Использован **throttle** через `requestAnimationFrame`, что даёт 60 fps даже при быстром скролле.

```js
let ticking = false;
window.addEventListener('scroll', () => {
    if (!ticking) {
        requestAnimationFrame(() => {
            update();
            ticking = false;
        });
        ticking = true;
    }
}, { passive: true });
```

`{ passive: true }` — подсказка браузеру, что обработчик не вызывает `preventDefault`, что ускоряет скролл.

### 9.3 navigation.js — меню и плавная прокрутка

- **Бургер-меню** — toggle класса `is-open` на `<nav>` + блокировка скролла на `<body>`.
- **Smooth scroll** — единственный делегированный обработчик на `document`, перехватывает клик по любой `<a href="#...">`.
- Вычитание `header.offsetHeight` обеспечивает корректное позиционирование при якорном переходе.

### 9.4 heroSlider.js — слайдер баннера

Собственная реализация без библиотек (~50 строк). Возможности:
- autoplay с настраиваемым интервалом;
- пагинация (точки сбоку);
- клавиатура ←/→;
- touch swipe;
- pause on hover.

```js
let startX = 0;
hero.addEventListener('touchstart', (e) => startX = e.touches[0].clientX, { passive: true });
hero.addEventListener('touchend', (e) => {
    const dx = e.changedTouches[0].clientX - startX;
    if (Math.abs(dx) > 50) dx > 0 ? prev() : next();
});
```

### 9.5 modal.js — модальные окна

Открытие/закрытие через data-атрибуты:

```html
<button data-modal-open="callback">Заказать звонок</button>
<div class="modal" id="modal-callback">...</div>
```

Реализованы:
- блокировка скролла фона;
- закрытие по Escape;
- закрытие по клику на backdrop;
- **focus management** (возврат фокуса на элемент, открывший модалку).

### 9.6 form.js — валидация и маска

**Маска телефона:** при вводе цифр форматирует в `+996 (XXX) XXX-XXX`. Использует регулярное выражение `/\D/g` для извлечения только цифр.

**Валидаторы** объединены в объект для масштабируемости:

```js
const validators = {
    name: (v) => v.trim().length < 2 ? 'Имя слишком короткое' : '',
    phone: (v) => !PHONE_REGEX.test(v) ? 'Некорректный номер' : '',
};
```

**Отправка** — имитация через `setTimeout` + `async/await`. В реальном проекте здесь был бы `fetch('/api/leads', { method: 'POST', body: ... })`.

### 9.7 scrollAnimations.js — IntersectionObserver

Самая современная API для отслеживания видимости. **Не использует** скролл-листенеры — браузер сам уведомляет когда элемент попадает в viewport.

```js
const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
        if (entry.isIntersecting) {
            entry.target.classList.add('is-visible');
            observer.unobserve(entry.target); // одноразовая анимация
        }
    });
}, { threshold: 0.15, rootMargin: '0px 0px -60px 0px' });
```

### 9.8 counter.js — анимация чисел

Принцип:
1. IntersectionObserver ловит появление `<span data-count="800">`.
2. `requestAnimationFrame` отрисовывает 60 кадров в секунду.
3. **Easing-функция** `easeOut = 1 - (1-t)³` даёт замедление к концу.
4. `toLocaleString('ru-RU')` форматирует числа с разделителями (800 → «800»).

## 10. Адаптивность

### 10.1 Стратегия — Desktop First

Базовые стили описаны для десктопа. Media queries уменьшают сложность вниз по контрольным точкам.

| Точка | Устройство | Что меняется |
|-------|-----------|--------------|
| ≤ 1200px | Малые десктопы | Уменьшение отступов |
| ≤ 992px | Планшеты | Бургер, 1-2 колонки в гридах |
| ≤ 768px | Большие мобильные | Уменьшение размеров шапки, кнопок full-width |
| ≤ 480px | Малые мобильные | 1 колонка везде, минимальные паддинги |

### 10.2 Гибкая типографика

Использован `clamp()` для жидких заголовков:

```css
.hero__title {
    font-size: clamp(38px, 6vw, 84px);
}
```

→ на мобиле минимум 38px, на десктопе максимум 84px, между ними — 6 % ширины окна.

### 10.3 Бургер-меню

При экране ≤ 992px:
- навигация уходит за правый край (`right: -100%`);
- появляется кнопка-бургер;
- при клике добавляется `.is-open` и блокируется скролл `<body>`.

## 14. Развёртывание и запуск

### 14.1 Требования

- Любой современный браузер (Chrome 90+, Firefox 88+, Safari 14+);
- Локальный HTTP-сервер (для работы ES-модулей нужен http://, не file://).

### 14.2 Запуск через Python

```bash
cd /Users/nazaramanturov/Desktop/js-project
python3 -m http.server 8000
```

Открыть в браузере: `http://localhost:8000`

### 14.3 Запуск через Node.js (опционально)

```bash
npx serve .
# или
npx http-server -p 8000
```

### 14.4 Деплой

Сайт деплоится **статически** на любой хостинг:
- Netlify (drag-n-drop папки);
- Vercel (`vercel deploy`);
- GitHub Pages (push в `gh-pages`);
- любой VPS с Nginx.

Никаких билд-шагов не требуется — браузер сам понимает ES-модули.
